# Regla: Arquitectura Modular

**Alcance:** Todo el proyecto (backend y frontend)

**Razón:** Facilita el mantenimiento, escalabilidad y separación de responsabilidades. Permite que los equipos trabajen de forma independiente en cada módulo.

**Aplicación actual:**
- Backend y frontend están completamente separados.
- El frontend está modularizado en carpetas (dashboard, ui, lib).
- El backend usa FastAPI y Pydantic para modularidad y validación.
